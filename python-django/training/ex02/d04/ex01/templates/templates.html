Default Template Engine:

Django's template engine follows a template-rendering approach, where templates are HTML files with embedded template tags and expressions. When a template is rendered, these tags and expressions are replaced with values from the context passed to the template.

Blocks:

Blocks in Django templates are used to define areas in a template that can be overridden in child templates. The block tag is used to define a block, and the extends tag is used in a child template to inherit from a parent template and optionally override specific blocks.

Loops (for ... in):

Django supports loops using the for item in list syntax. It allows you to iterate over a list or queryset and display content dynamically.

If Control Structures:

Django templates support conditional logic using if condition and else tags.

Display of Passing Context Variables:

Context variables, usually passed from a Django view, are accessed in templates using double curly braces ({{ variable_name }}).
